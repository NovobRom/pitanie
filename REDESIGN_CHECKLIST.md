# Redesign Checklist — pitanie.vercel.app

> Цель: превратить статичную "таблицу из 90-х" в современное, удобное приложение для двоих.
> Аудитория: Роман + Лиза (личное использование, мобильный телефон — основной способ просмотра).

---

## 1. Дизайн-система и общий стиль

- [ ] **Выбрать UI-концепцию**: мягкий, тёплый дизайн (не корпоративный). Ориентир — приложения типа Mealime, Yazio, Samsung Health.
- [ ] **Цветовая палитра**: уйти от серо-голубого Tailwind-дефолта. Создать тёплую палитру (sage green, warm beige, мягкий оранжевый для акцентов). Роман = синий акцент, Лиза = розовый — оставить, но сделать мягче.
- [ ] **Типографика**: заменить `Arial, Helvetica, sans-serif` и Geist на что-то более "живое" — например Inter или Nunito (есть в Google Fonts). Настроить размеры и weight для иерархии.
- [ ] **Иконки**: убрать Font Awesome (CDN из layout.tsx). Перейти на `lucide-react` или `react-icons` — легче, modern, tree-shakeable.
- [ ] **Spacing-система**: выровнять отступы по единой сетке (4/8/16/24/32/48px).
- [ ] **border-radius**: увеличить скругления, сделать мягче (16-20px для карточек).
- [ ] **Тени**: заменить `shadow-md`/`shadow-lg` на более мягкие, еле заметные тени (soft shadow).

---

## 2. Layout и навигация

- [ ] **Mobile-first**: весь дизайн строить от мобильного экрана. Сейчас на мобильном всё сжато.
- [ ] **Sticky header**: зафиксировать шапку с именами/калориями наверху при скролле — всегда видно свою норму.
- [ ] **Навигация по дням**: добавить горизонтальные табы (Пн Вт Ср...) или swipeable-карточки вместо длинного скролла через все 7 дней.
- [ ] **Якорные ссылки**: если оставить скролл — добавить быстрые ссылки на каждый день.
- [ ] **Bottom navigation (мобильная)**: внизу экрана 3 таба — "Меню", "Продукты", "Покупки".

---

## 3. Header (шапка)

- [ ] **Редизайн карточек**: вместо плоских `bg-blue-50` блоков — красивые gradient-карточки с аватарками (или инициалами) и прогресс-баром дневной нормы.
- [ ] **Убрать кнопку "Распечатать"**: перенести в меню (три точки) или убрать совсем — основной юзкейс это мобильный телефон.
- [ ] **Метаданные в layout.tsx**: заменить `title: "Create Next App"` на нормальное название и описание. Добавить `lang="ru"`.

---

## 4. Карточки дней (DailyMenu)

- [ ] **Вынести данные из JSX**: сейчас все 7 дней захардкожены в одном компоненте (367 строк!). Вынести в `data/menu.ts` — массив объектов `{ day, grain, meals: [...] }`.
- [ ] **Компонент MealCard**: создать переиспользуемый компонент для одного приёма пищи.
- [ ] **Компонент DayCard**: одна карточка дня, принимает пропсы.
- [ ] **Цветовое кодирование**: каждый приём пищи — свой мягкий цвет (завтрак — тёплый жёлтый, обед — зелёный, ужин — лавандовый).
- [ ] **Заголовки дней**: вместо чёрных bg-gray-800 полос — мягкие градиенты или цветные акценты сверху карточки.
- [ ] **Collapse/Expand**: возможность свернуть/развернуть день или приём пищи (особенно на мобильном).
- [ ] **Воскресенье (читмил)**: сделать особенной — яркая карточка, иллюстрация, отличается от будней.

---

## 5. Таблица продуктов (CheatSheet)

- [ ] **Убрать табличный вид**: заменить `<table>` на карточки продуктов (каждый продукт — маленькая карточка с иконкой, граммовкой для обоих).
- [ ] **Визуальные иконки продуктов**: заменить эмодзи на стильные SVG/иллюстрации или хотя бы сделать эмодзи крупнее и выразительнее.
- [ ] **Убрать border-l-8**: толстая зелёная полоса слева выглядит устаревшей.

---

## 6. Список покупок (ShoppingList)

- [ ] **Интерактивные чекбоксы**: сейчас чекбоксы не сохраняют состояние. Добавить `localStorage` для сохранения отмеченных.
- [ ] **Анимация при отметке**: зачёркивание текста + лёгкая анимация.
- [ ] **Счётчик**: "Куплено 5 из 12".
- [ ] **Тёмный фон**: заменить тёмный `bg-gray-800` блок — он слишком контрастный. Сделать в общем стиле с мягким фоном.
- [ ] **Кнопка "Сбросить всё"**: очистить все чекбоксы.

---

## 7. Технический долг

- [ ] **layout.tsx**: удалить `<link>` на Font Awesome CDN из `<head>`.
- [ ] **layout.tsx**: обновить `metadata` — title, description, og:image.
- [ ] **layout.tsx**: `lang="en"` -> `lang="ru"`.
- [ ] **globals.css**: `@tailwind` директивы устарели для Tailwind v4. Проверить и обновить.
- [ ] **Компоненты**: Tailwind v4 (@tailwindcss/postcss ^4) — убедиться, что конфигурация корректна.
- [ ] **Данные**: вынести все захардкоженные данные в отдельные файлы (`data/menu.ts`, `data/products.ts`, `data/shopping.ts`).

---

## 8. Приятные бонусы (nice-to-have)

- [ ] **PWA (Progressive Web App)**: добавить manifest.json + service worker. Позволит "установить" сайт на телефон как приложение и пользоваться офлайн.
- [ ] **Тёмная тема**: auto-переключение по системным настройкам телефона.
- [ ] **"Сегодня"**: автоматически подсвечивать текущий день недели.
- [ ] **Переключатель "Показать только меня"**: фильтр — видеть только свою порцию (Роман или Лиза).
- [ ] **Анимации**: мягкие transitions при открытии карточек, hover-эффекты.

---

## Порядок работы (рекомендация)

1. Дизайн-система (цвета, шрифты, spacing) — globals.css + tailwind config
2. Рефакторинг данных (вынести из JSX в data/)
3. Создать переиспользуемые компоненты (MealCard, DayCard)
4. Редизайн Layout + Header + навигация по дням
5. Редизайн карточек дней
6. Редизайн таблицы продуктов
7. Редизайн списка покупок + localStorage
8. Технический долг (meta, lang, иконки)
9. PWA + бонусы
